Answer for Q 1.1 Expected output in Turbo C++ 
sizeof near*  n1: 2
              n2: 2
sizeof  far*  f1: 4
              f2: 2
sizeof huge*  h1: 4
              h2: 2
sizeof char* ch1: 2
             ch2: 1
sizeof char* ch3: 2
             ch4: 2
near, far, and huge are non-standard keywords used only in Turbo C/Turbo C++.
They were introduced to handle the 16-bit segmented memory architecture of Intel 8086 processors.

Memory in Turbo C is divided into segments of size 64 KB (65536 bytes).
Each segment contains offsets from 0 to 65535.

Concept Explanation
near pointer 2 bytes. Stores only OFFSET within current segment.
far pointer 4 bytes. Stores SEGMENT + OFFSET, can access any memory segment.
huge pointer 4 bytes. Like far pointer but normalized for large data traversal.

Important clarification:
sizeof(pointer) does NOT represent memory being pointed to.
It only represents the size of the pointer variable itself.

Example:
- near pointer = 2 bytes (offset only)
- far pointer = 4 bytes (segment + offset)
- huge pointer = 4 bytes

Correct memory model:
- 1 segment = 65536 bytes (64 KB)
- near pointers operate within a single segment only
- far pointers can access multiple segments in full memory space

Final conclusion:
The output of sizeof() reflects only pointer sizes, not actual memory usage of data.
Memory in Turbo C is segmented, not arranged as small byte boxes.
There is no concept of total bytes limiting address rangeish based on sizeof calculations.
